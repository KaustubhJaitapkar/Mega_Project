All Roles — Hackathon Scope

Only features scoped for a college hackathon MVP. Green = Participant · Blue = Organiser · Orange = Judge · Purple = Mentor

◆ Participant
🃏
Talent Profile
Identity
  • → Name, skills, role preference (dev / design / PM)
  • → GitHub link + brief bio
  • → "Looking for team" toggle — shows up in public discovery feed
👥
Team Formation
Team
  • → Create a team or join via invite code
  • → View other participants who are looking for a team
  • → Team lead can accept / reject join requests
📦
Final Submission Portal
Submission
  • → Submit GitHub Repo URL + Live Demo URL
  • → Project Health Check: pings both links to verify they're live before deadline
  • → Upload pitch deck (PDF/PPT) and add tech stack tags
  • → Draft save + resubmissions allowed until deadline lock
🙋
Raise a Hand (Mentor Help)
Support
  • → Tag-specific help requests: #ReactError, #FirebaseIssue, #IdeaBlock
  • → Request appears in assigned mentor's live Help Queue
  • → Status: Pending → Mentor On Way → Resolved
📱
QR Code + Logistics
On-Ground
  • → Personal QR code in dashboard — volunteer scans for check-in
  • → Scanned again for "Lunch Redeemed" and "Swag Collected" — prevents abuse
  • → Attendance state: Registered → Checked In → Submitted
◆ Organiser
🗓️
Event Setup + Timeline
Setup
  • → Custom event page: banner, logo, tagline, tracks/themes
  • → Visual event timeline shown to all roles (pre-event + hackathon day)
  • → Team size limits and eligibility rules (college, year, branch)
  • → Public / invite-only / college-email-gated visibility
👥
Add Mentors + Judges
People
  • → Invite mentors and judges via email — they get limited-access portal
  • → Manually assign mentors to teams
  • → Teams notified of their assigned mentor before hackathon day
📡
Announcement Broadcast
Communications
  • → Push alerts to all participant dashboards (in-app notifications)
  • → Pinned announcements on event dashboard visible to all roles
🔒
Kill Switch
Control
  • → Lock / unlock submissions globally or per-team
  • → Extend deadline for specific teams (with audit log)
📊
Basic Dashboard
Analytics
  • → Registration count, team count, submission count
  • → Live check-in count and meal/swag redemption stats
  • → CSV export of participants
◆ Judge
🖥️
Split-View Evaluation
Core UI
  • → GitHub + Live Demo links on left, rubric sliders on right
  • → Judges must add notes explaining each score — anti-bias proof
  • → Notes are printable as a bias-audit document post-event
🙈
Blind Folder
Anti-Bias
  • → Anonymised project names — prevents college-name bias
  • → Organiser toggles blind mode on/off per judging round
  • → Names revealed only after all scores are locked
🔏
Review & Seal (One-Way Submit)
Integrity
  • → "Review & Seal" button — irrevocably locks marks into database
  • → Confirmation modal: shows final scores + notes before sealing
  • → Sealed scores are timestamped and immutable — full audit trail
◆ Mentor
🎫
Live Help Queue
Real-time
  • → See all "Raised Hands" from assigned teams in real time
  • → Filtered by tag: #ReactError, #FirebaseIssue, #DesignHelp
  • → Accept a request → status changes for team to "Mentor On Way"
👀
Team Overview
Monitoring
  • → Read-only view of all assigned teams' submission status
  • → See team name, project title, and submission state at a glance
✅
Scope Note: Sponsor role, XP/badges, ghost slots recommendation engine, AI Idea Validator, and GitHub activity monitoring have been moved to Future Scope. These are high-value but non-essential for a first college hackathon. Build the core loop first — teams, submissions, judging, QR. Ship with this, validate, then layer in more.
Event Lifecycle — 3 Phases

What happens from setup to post-event, mapped to hackathon scope.

Phase 01
Pre-Hackathon Setup
Organiser-led
🎛️
Organiser creates the event — sets title, banner, timeline, tracks, team size limits, eligibility rules, visibility mode.
⚖️
Organiser configures the rubric — sets judging criteria with weightings (e.g. Innovation, Technical, Impact, Presentation).
👥
Organiser invites mentors and judges via email. They receive portal logins with role-limited access.
📋
Participants register — create profile, join or create teams, invite teammates via code.
🔔
Mentor assignment notification — participants are informed of their assigned mentor before hackathon day.
Phase 02
Hackathon Day (Live)
All roles active
📱
QR check-in at venue — volunteer scans participant QR, marks as "Checked In". Same QR used for lunch and swag redemption.
⏱️
Live event timeline + countdown timer visible to all — time-specific milestones shown on the event dashboard.
📡
Organiser broadcasts updates — announcements pushed to participant dashboards.
🙋
Participants raise hands — tagged help requests appear in mentor's live Help Queue.
🖥️
Judge rounds begin — judges use split-view + mandatory notes. Blind folder optionally active.
📦
Final submission portal — teams submit GitHub + Live URL. Health Check pings links before deadline. Kill Switch available to organiser.
🔏
"Review & Seal" locks all marks — scores are immutable and timestamped. Final leaderboard generated.
Phase 03
Post-Hackathon
Wrap-up
📜
Digital certificates auto-generated — downloadable PDFs with participant name, team name, rank (Winner / Runner-Up / Participant). LinkedIn-shareable link.
📝
Feedback survey — participants rate mentors, judges, food, and organisation. Data exported for organiser.
🖨️
Judge bias-audit report — printable PDF of all judge notes per project. Proof that scoring was justified.

How to Build This with AI ✦

You don't need a team of 10 engineers. Here's exactly which AI tools to use, when to use them, and what prompts to give them.

The Core Principle

Modern AI tools have made it possible for a solo founder or small 2-person team to build a production-grade platform. The key is knowing which AI does what. Think of it like having a team: Claude is your architect + senior developer, Cursor/Copilot is your coding pair-programmer, v0 is your UI designer, and Claude Code is your autonomous devops engineer. You are the product manager and decision-maker.

🤖
Claude (claude.ai)
Architect · System Designer · Senior Dev
Use for planning, architecture decisions, complex backend logic, database schema design, API structure, writing full feature specs, and debugging hard problems. Best tool for thinking through tradeoffs. Great for writing Prisma schemas, auth flows, and complex queries.
⌨️
Claude Code
Autonomous Engineer · CLI Agent
Use for running entire features end-to-end in your codebase. Tell it: "Implement the QR check-in system — create the API route, the DB model, and the participant dashboard QR display." It reads your files, writes code, runs tests, and fixes errors without you copy-pasting.
🖱️
Cursor / GitHub Copilot
Pair Programmer · Daily Coding
Your day-to-day coding environment. Cursor has full codebase context. Use Cmd+K to edit existing functions inline. Use Composer for multi-file changes. Best for iterative work: "Add validation to this form", "Refactor this to use React Query", "Add error handling here."
🎨
v0 by Vercel
UI Designer · Component Generator
Use for generating UI components from descriptions. "Build a split-view judge evaluation panel — GitHub/demo iframe on left, rubric sliders on right, notes textarea at bottom." Outputs production-ready Next.js + ShadCN + Tailwind code. Best starting point for every new screen.
🗄️
Supabase + AI Assist
Database · Auth · Real-time
Supabase is Postgres + Auth + Real-time + Storage in one. Use its built-in AI to generate SQL migrations, RLS policies, and triggers. Their SQL editor has AI autocomplete. Generate your entire schema by describing it in plain English.
🚀
Lovable / Bolt.new
Full-Stack Scaffolding · Fast Prototyping
Use at the very start to generate the entire initial scaffold from a description. "Build a hackathon platform with participant/organiser roles, team management, and a project submission form." Get 70% of the boilerplate for free in hours, then refine with Cursor + Claude.

Future Scope 🔮

These are all valuable features — just not for the first build. Park them here, validate your MVP first, then come back. Each one is well-defined enough to pick up when the time is right.

🔮
Why Future Scope? These features are either technically complex, require third-party integrations, or are only valuable at scale. Build the core hackathon loop first — teams, submissions, judging, QR. Once real users validate that, layer these in one at a time.
Team Discovery & Matching
👻 Team Discovery
Ghost Slots + Opportunity Gallery
Teams advertise specific open roles — "Looking for: 1× UI/UX Designer" — in a public gallery. Solo participants browse and "Ping" teams directly. WhatsApp contact bridges digital to real life.
Phase 3
🧠 AI Matching
Recommendation Engine (Perfect Match Feed)
Algorithm ranks teams by compatibility — team has 2 backend devs? Suggests frontend match. v1: tag-based weighted scoring. v2: cosine similarity with pgvector. Prevents ghost requests with mutual-interest counters.
Phase 3 → 4
Organiser Power Features
📊 Analytics
Command Center (Real-Time Heatmap)
Real-time heatmap of team registrations, skill distributions, registration funnel (signup → team formed → submitted), and college/branch/year breakdowns with CSV export.
Phase 3
⚖️ Judging
Dynamic Rubric Builder
Drag-and-drop rubric creation with weighted percentages that must sum to 100%. Configure number of judging rounds. Assign projects to judges in blind or open mode. Currently handled with a simpler static rubric config.
Phase 3
📡 Communications
Scheduled Broadcast + Discord/WhatsApp Integration
Schedule-based auto-announcements (e.g. "1 hour to submission deadline"), integrated Discord channel broadcast, and WhatsApp group notifications. Basic in-app announcements ship in v1 — external integrations come later.
Phase 3
🏛️ Multi-College
Multi-College Event Umbrella
Support events where multiple colleges collaborate — each college has a local coordinator managing their institution's registrations, while a central organiser sees everything. Common in inter-college fests.
Phase 4
Judge Power Features
🎪 Live Round
Presentation Queue (Stage View)
Stage View showing which team is presenting now and who's next. Judges prep by viewing earlier round scores. Multiple judging rounds supported with cumulative scoring. Ships after basic judging is validated.
Phase 3
Mentor Power Features
📊 GitHub Monitoring
GitHub Commit Activity Monitor
Mentors see GitHub commit activity for assigned teams. Auto-alert after 6 hours of no commits — mentor can proactively check in before the team realises they're stuck. Requires GitHub API integration and polling.
Phase 3
🗓️ Async / Online
Office Hours Scheduler
Calendar for mentors to mark available 1-on-1 slots. Teams book 20-min brainstorming sessions. Auto-generates Google Meet link on booking. Primarily relevant for online hackathons.
Phase 4
Sponsor Portal + Gamification
🏪 Branding
Sponsor Booth
Dedicated page with sponsor logo, bio, and sponsor-specific prize tracks (e.g. "Best use of [Their API]"). Sponsor can post workshop announcements visible to all participants.
Phase 3
🎯 Recruitment
Talent Export for Sponsors
Sponsors browse opted-in participant Talent Profiles, filter by skills/year/college/project, and export to CSV. Direct pings to notify participants of sponsor workshops or opportunities.
Phase 3
🏅 Gamification
XP + Badges + Public Developer Portfolio
Every hackathon earns XP. Badges for milestones: first hack, team leader, mentor's pick, 3-peat finisher. Participants build a public profile shareable on LinkedIn. Drives long-term platform return rate.
Phase 3
AI-Powered Features
🤖 AI
Idea Validator (Claude/GPT)
When a team submits their idea (before final submission), run it against an LLM — checks originality, feasibility, and gives improvement suggestions. Judges also see the AI report alongside the project. Requires Anthropic/OpenAI API integration.
Phase 4
Post-Hackathon
🤝 After the Hack
Post-Hackathon Collab Board
After results, teams can mark their project as "open to continue". Other participants or the public can express interest. Turns hackathon prototypes into real ongoing projects. Builds long-term platform stickiness.
Phase 4
📱 Mobile
Mobile App (React Native)
A dedicated mobile app for on-ground hackathon experience — QR scanning, help queue, live timeline, and push notifications. The web app handles mobile use for v1; a native app unlocks better on-ground UX at scale.
