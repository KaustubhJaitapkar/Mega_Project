# 3-Minute Hackathon Pitch Script

## 0:00 - 0:20 Problem

"Today, users facing `<problem>` lose `<time/money/opportunity>` because existing solutions are `<limitation>`. This is a real and frequent issue for `<target users>`."

## 0:20 - 0:40 Why Existing Options Fail

"Current tools are either too complex, too expensive, or not designed for this exact workflow. Users end up using manual workarounds."

## 0:40 - 1:50 Live Demo

Show exactly one main user journey:
1. Input
2. Processing
3. Output
4. Value delivered

Narration tip: explain result in user language, not engineering language.

## 1:50 - 2:20 Technical Depth

"Under the hood, we built `<architecture>` using `<stack>`, optimized for `<speed/reliability/accuracy>`. We handled `<hard problem>` through `<approach>`."

## 2:20 - 2:45 Impact

"This can help `<user group>` by `<measurable result>`. In pilots or expected usage, this means `<metric>`."

## 2:45 - 3:00 Close

"Hackmate2 is not just a concept. It works today, solves a real pain point, and is ready to scale with `<next step>`."

## Q&A Prep

- Why this over existing tools?
- What if your external dependency fails?
- How do you scale to 10x users?
- What is your validation plan?
